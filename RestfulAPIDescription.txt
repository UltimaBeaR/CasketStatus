Публичный RESTful API сайта status.casket.com для разработчиков мобильных приложений

Запрос
Для получения текущего состояния работы сервиса casket.com, нужно отправить REST запрос вида
GET /api/status HTTP/1.1 на хост status.casket.com по протоколу HTTP
Пример ссылки в адресной строке браузера, для выполнения запроса из браузера: http://status.casket.com/api/status

Результат
В случае успеха будет результат вида HTTP/1.1 200 OK
Тип содержимого: application/json
Содержимое - строка в JSON формате

При отправке без параметров, минимальный результат будет таким
{"code":СТАТУС_КОД }
где code - код текущего статуса сервиса.

code может принимать следующие значения:
0: Сервис работает, технические работы не запланированы
1: Серис не работает, ведутся технические работы
2: Сервис работает, запланированы технические работы

В случае, если code принимает значение 2, дополнительно в результате будет указана дата запланированных технических работ, например
{"code":2,"scheduledWorkDate":"2016-06-24T23:55:46"}

Опционально может использоваться параметр description, принимающий значения true/false, например
GET /api/status?description=true HTTP/1.1
В случае значения true дополнительно в результате будут указаны текстовые описания статуса text (Краткое описание) и detailedText (Расширенное описание), например
{
	"code":2,
	"scheduledWorkDate":"2016-06-24T23:55:46",
	"text":"Запланированы работы",
	"detailedText":"Все работает штатно, но на 24.06.2016 23:55:46 запланированы работы технического характера. Сервис будет недоступен на некоторое время."
}
